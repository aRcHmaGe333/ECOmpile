# OpenAI Handoff Package

This document defines what ECOmpile sends in outreach and what remains internal to OpenAI.

## Package Objective

Provide a clear split:
1. externally provable ECOmpile evidence (already in place),
2. internal-only validation steps OpenAI can run in-house,
3. commercial guardrail for attribution and compensation.

## External Package Components

Generated by:

```bash
python sdk/examples/openai_handoff_compiler.py --case-file cases/2026-03-04_unknown-contact-sid-removal/conversation.md --spec-file cases/2026-03-04_unknown-contact-sid-removal/openai_handoff_spec.json --strict
```

Outputs:
1. `behavioral_evidence.json`
2. `internal_trace_contract.json`
3. `SID_REMOVE_SYSTEM_WIDE_KNOWN.kernel.candidate.md`
4. `handoff_summary.md`

## Validation Boundary

1. Behavioral evidence can be extracted from logs.
2. True model-internal introspection cannot be extracted from logs alone.
3. True introspection requires in-house telemetry for:
- intent extraction events,
- kernel shortlist/match scoring,
- stop-condition decisions,
- primitive emit events,
- forbid-branch pruning events.

## Commercial Guardrail

Compensation and attribution terms are agreed in writing before internal execution, integration, derivative use, or deployment.
